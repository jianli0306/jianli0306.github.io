    Docker 和虚拟机的区别

    Docker 的容器(containers)、镜像(images)、仓库(Registry)

    Docker 镜像是一个只读的模板，用来创建容器，容器时 Docker 运行的实例，提供了一个独立的可移植的环境

    Docker 仓库是用来存储 Docker 镜像的地方，最流行和常用的仓库就是 Dockerhub（公共的 Docker 仓库），镜像共享和复用

    服务端的 docker daemon 是服务端的 docker 守护进程，客户端和服务端的通信在此。docker daemon 接收到客户端的各种请求后进行处理，将结果返回给客户端

    容器化：将应用程序打包成容器，在容器中运行应用程序

    容器化三个步骤

    1. 创建一个 Dockerfile，来告诉 Docker 构建应用程序镜像的步骤和需要的配置
      1. Dockerfile 是一个文本文件，无任何扩展名，包含一条条指令
      2. 包括精简版的操作系统，如 Alpine
      3. 应用程序的运行时环境，如 NodeJS（JavaScript 不在浏览器运行时的环境）、Java、Python
      4. 应用程序，如 SpringBoot 打包好的 jar 包
      5. 应用程序的第三方依赖库或者包
      6. 应用程序的配置文件、环境变量等
    2. 使用 Dockerfile 来构建镜像
    3. 使用镜像创建和运行容器

    构建 Docker 镜像的例子：运行 JavaScript 命令

    1. 新建项目（镜像名字），要运行的 index.js 文件到项目根目录
    2. 新建 Dockerfile 文件
      1. 指定基础镜像，一般会有包含操作系统和运行环境（Step1 和 Step2）
    2. 复制文件到容器根目录中 COPY source dest

    3. 运行应用程序 CMD
    1. 构建镜像

    docker build -t hello-docker .

    其中 hello-docker 即镜像名，.代表 dockerfile 文件所在的当前目录

    1. 使用docker images或docker image ls来查看我们的镜像
    2. 运行镜像

    docker run hello-docker

    1. 因为 Dockerfile 文件中有运行应用程序的命令，所以运行镜像后我们的程序运行

    以后可以只传镜像和运行镜像即可。也可以将镜像上传到 DockerHub 或仓库：使用 docker pull 镜像地址

    Docker 部署 MySQL

    使用Docker安装 MySQL，仅仅需要一步即可，在命令行输入下面的命令：

    解读：

    • docker run -d ：创建并运行一个容器，-d则是让容器以后台进程运行
    • --name`` mysql : 给容器起个名字叫mysql，你可以叫别的
    • -p 3306:3306 : 设置端口映射。
      • 容器是隔离环境，外界不可访问。但是可以将宿主机端口映射容器内到端口，当访问宿主机指定端口时，就是在访问容器内的端口了。
      • 容器内端口往往是由容器内的进程决定，例如MySQL进程默认端口是3306，因此容器内端口一定是3306；而宿主机端口则可以任意指定，一般与容器内保持一致。
      • 格式： -p 宿主机端口:容器内端口，示例中就是将宿主机的3306映射到容器内的3306端口
    • -``e`` TZ=Asia/Shanghai : 配置容器内进程运行时的一些参数
      • 格式：-e KEY=VALUE，KEY和VALUE都由容器内进程决定
      • 案例中，TZ``=Asia/Shanghai是设置时区；MYSQL_ROOT_PASSWORD=123是设置MySQL默认密码
    • mysql : 设置镜像名称，Docker会根据这个名字搜索并下载镜像
      • 格式：REPOSITORY:TAG，例如mysql:8.0，其中REPOSITORY可以理解为镜像名，TAG是版本号
      • 在未指定TAG的情况下，默认是最新版本，也就是mysql:latest

    当我们执行命令后，Docker做的第一件事情，是去自动在仓库中搜索并下载了MySQL，然后会自动运行MySQL，我们完全不用插手

    这种安装方式你完全不用考虑运行的操作系统环境，它不仅仅在CentOS系统是这样，在Ubuntu系统、macOS系统、甚至是装了WSL的Windows下，都可以使用这条命令来安装MySQL

    Docker会自动搜索并下载MySQL。注意：这里下载的不是安装包，而是镜像。镜像中不仅包含了MySQL本身，还包含了其运行所需要的环境、配置、系统级函数库。因此它在运行时就有自己独立的环境，就可以跨系统运行，也不需要手动再次配置环境了。这套独立运行的隔离环境我们称为容器

    Docker 基础命令

    注意 docker run 是启动并运行一个容器，每次执行这个命令都是启动一个新容器

    而 docker stop 则是停止了容器中的进程，如 mysql 容器中的 mysql 进程，docker start 则是启动容器中的进程。在这两个命令中，容器都是一直存在的，停止和启动的都是容器中的进程

    docker ps 用来查看 docker 状态（默认只查看运行的容器），docker ps -a 查看运行和停掉的容器

    docker exec 可以进入容器内部，因为 docker 容器都是隔离的：

    -it 表示可以交互的进入，bash 表示进入后的交互方式

    如进入 mysql 容器，然后连接 mysql

    docker inspect 容器名可以查看容器对应的信息

    docker rmi删除镜像

    docker rm 删除容器

    docker rm -f 强制删除容器 （不会删除一些挂载目录之类的信息)
